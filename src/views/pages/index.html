Quick Start

1. Install Dependencies

npm install

Install all required dependencies including Vite, SCSS compiler, and build tools.

2. Start Development

npm run dev

Start the development server with hot reload and automatic page generation.

Available Commands

npm run dev Development

Start development server with hot reload

npm run build Production

Build optimized production bundle

npm run build:public Public Assets

Build and optimize public assets (fonts, images)

npm run build:watch Watch Mode

Build in watch mode for development

npm run generate:pages Utility

Generate pages.json from HTML files

npm run generate:pages:watch Utility

Watch and auto-generate pages.json

npm run generate:components Components

Generate clean HTML files for all components

npm run generate:docs Documentation

Generate component documentation (markdown & HTML)

npm run figma:sync Figma Sync

Full sync of design tokens and components from Figma

npm run figma:tokens Figma Tokens

Sync color and typography styles from Figma only

npm run figma:scaffold Figma Scaffold

Scaffold new component folders based on Figma library

npm run test Testing

Run JavaScript tests in watch mode

npm run test:run Testing

Run JavaScript tests once

npm run test:ui Testing UI

Run tests with browser interface

Powerful templating with layouts, includes, and macros for reusable components.

Modern CSS with variables, mixins, and component-based styling architecture.

Asset Optimization

Testing Framework

Comprehensive JavaScript testing with Vitest for code quality and validation.

Figma Integration

Automated design-to-code workflow with token syncing and component scaffolding.

Key Features

Auto Page Generation

Automatically generates pages.json from HTML files in the pages directory.

Component HTML Export

Generates clean, formatted HTML files from Nunjucks components for backend integration.

Asset Optimization

Automatic WebP conversion, font format generation, and image compression.

Responsive Design

Bootstrap 5.3 integration with custom responsive utilities and mixins.

Component Architecture

Modular component system with separate JS/SCSS files for each component.

JavaScript Testing

Comprehensive testing framework with Vitest featuring automatic component discovery and code quality validation.

Figma Sync System

Automated syncing of design tokens and component scaffolding to maintain design-dev consistency.

Changelog

v1.5.0 January 2026
Added
Figma Design System Integration
  • Design Token Sync: Automated extraction of color and typography styles from Figma via REST API
  • SCSS Generation: Automatically generates _settings.colors.figma.scss and _settings.fonts.figma.scss
  • Component Scaffolding: New figma:scaffold command to auto-generate component folders with NJK, SCSS, and JS templates
  • Configuration System: Dedicated figma.config.js for mapping tokens and defining scaffolding rules
  • Environment Security: Secure credential management using .env files
  • Flexible Sync: Support for partial sync (tokens only or scaffold only) and specific component scaffolding
v1.4.0 January 2026
Added
Component Documentation Generator
  • Automated Documentation: New script scripts/generate-component-docs.js automatically generates comprehensive component documentation
  • Dual Output Formats: Generates both Markdown (docs/component-documentation.md) and interactive HTML (docs/component-documentation.html) versions
  • Entity Extraction: Automatically identifies inputs, buttons, links, images, headings, and data attributes from components
  • Layout Detection: Detects grid columns, flex layouts, responsive variants, BEM modifiers, and size variations
  • Component Relationships: Maps nested components and reverse "Included in" relationships showing component dependencies
  • Interactive HTML: Sticky sidebar navigation with search, active section highlighting, and smooth scrolling
  • Pattern Recognition: Identifies forms, carousels, modals, grids, accordions, and table structures
  • Page Mapping: Bidirectional mapping between components and pages (72 components, 42 pages)
  • Usage Statistics: Shows most-used components, unused components, and complexity metrics
  • Build Integration: Runs automatically during npm run generate:all
  • New Command: npm run generate:docs for manual documentation generation
v1.3.0 September 2025
Added
JavaScript Testing Framework with Vitest
  • Testing Framework: Integrated Vitest for JavaScript validation and testing
  • Test Commands: npm test, npm run test:run, and npm run test:ui
  • Consolidated Testing: Single test files validate multiple source files efficiently
  • Test Coverage: All JavaScript files in src/js/ and src/views/components/
  • Dynamic Discovery: Automatically finds and tests new components without manual updates
  • Validation Features: File existence, syntax checking, import/export validation, and code quality rules
  • No Individual Files: Organized testing approach without cluttering source directories
v1.2.0 Previous Release
Added
Component HTML Generation System
  • New Script: scripts/generate-components.js - Converts Nunjucks components to clean HTML
  • New Commands: npm run generate:components and npm run generate:all
  • Output Location: Components exported to public/components/ directory
  • HTML Formatting: Proper 2-space indentation and structured output for backend developers
  • Manifest Generation: Auto-generated metadata file with component information
  • Build Integration: Component generation now runs automatically during build process
v1.1.0 Previous Release
Added
Dynamic Pages.json Generation
  • Auto-generation of pages.json from HTML files in pages directory
  • Watch mode for development with npm run generate:pages:watch
  • Build process integration for seamless development workflow
